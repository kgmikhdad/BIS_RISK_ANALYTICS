# Model Governance Notes

Analytical prototypes should clearly state their purpose, assumptions, data sources, limitations, validation checks, and intended use. A prototype should not be described as production grade unless it has passed appropriate model validation, code review, performance testing, access-control review, and business-owner approval.

For AI-assisted tools, governance should include document-source control, retrieval quality checks, traceability of cited evidence, prompt/version control, human review, and monitoring for hallucination or unsupported output.

For Monte Carlo and agent-based simulations, governance should include sensitivity analysis, random seed control, parameter documentation, stability checks, and comparison against simpler benchmark models.
